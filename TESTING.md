# Testing Workflow

This repository uses three layers of tests:

- Fast unit and invariant tests: numerical helpers, reproducibility, conservation laws, and edge cases.
- Regression tests: deterministic one-step collision outputs compared against stored baselines.
- Verification tests: full-scale physical relaxation histories for the canonical Fig. 4-6 cases.

## Running Tests

Install the development dependencies:

```bash
pip install -e ".[dev]"
```

Run the fast suite:

```bash
pytest -q -m "not verification"
```

Run the full-scale verification suite:

```bash
pytest -q -m verification
```

Run everything:

```bash
pytest -q
```

Run the merge-time compatibility check against `main`:

```bash
python scripts/main_compatibility_report.py
```

## Baseline Files

Stored regression baselines live in `tests/data/`:

- `collision_unequal_weight_small_v1.npz`
- `relaxation_reduced_v1.npz`
- `fig4_bundle_v1.npz`
- `fig5_bundle_v1.npz`
- `fig6_bundle_v1.npz`

Stored rendered figure baselines live in `tests/data/figures/`:

- `fig4_bundle_v1.png`
- `fig5_bundle_v1.png`
- `fig6_bundle_v1.png`

The Fig. 4-6 bundle files are derived from the six runs encoded in `utilities/nanbu_figure_cases.py`.
They are the baseline for checking that the current branch still reproduces the same canonical scenarios that the original `main` example script intended to run.
By default these bundle baselines are generated at the original `main` figure scale.
The rendered PNG baselines are secondary visual artifacts for manual inspection. The official code-health baseline is the seeded numerical history stored in the `.npz` files.

These files are generated by:

```bash
python scripts/generate_baselines.py
python scripts/render_baseline_figures.py --figure-scale full --output-dir tests/data/figures
```

Only the `.npz` files are used for automated regression checks. The `.png` files are kept so a human can quickly compare the current relaxation curves against the approved baseline.

To regenerate reduced figure baselines for quick local experiments only:

```bash
python scripts/generate_baselines.py --figure-scale reduced
```

To compare regenerated baselines against the checked-in reference set:

```bash
python scripts/diff_baselines.py tests/data /path/to/new/baselines
```

## Baseline Update Policy

Do not regenerate baselines as part of normal refactoring.

Regenerate baselines only when:

- the physical model is intentionally changed
- the numerical method is intentionally changed
- the existing baseline is proven wrong

When baselines change:

1. run `python scripts/generate_baselines.py`
2. run `python scripts/render_baseline_figures.py --figure-scale full --output-dir tests/data/figures`
3. run `pytest -q`
4. run `python scripts/main_compatibility_report.py`
5. describe in the commit or PR why the physical result changed
6. mention which baseline `.npz` files changed and why that change is expected
7. if the visual appearance changed materially, mention the corresponding `.png` updates as secondary artifacts

If code changes are supposed to be behavior-preserving, the baseline files should not change.
